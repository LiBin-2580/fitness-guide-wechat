## 二、技术选型

| 层级 | 技术 | 选型理由 |
|------|------|----------|
| 前端 | 微信原生小程序 | 课设要求，学习成本低 |
| 后端 | Node.js + Express | 前后端统一语言，生态丰富 |
| 数据库 | MySQL 8.0 | 关系型数据，课设常见 |
| 认证 | JWT | 无状态认证，实现简单 |
| 密码加密 | bcrypt | 安全性高，业界标准 |

## 三、模块职责

| 模块 | 职责 |
|------|------|
| 认证模块 | 微信登录、Token管理、权限校验 |
| 计划模块 | 训练计划CRUD、动作管理 |
| 记录模块 | 训练打卡、记录查询 |
| 数据模块 | 身体数据录入、趋势计算 |
