## 一、设计风格定义

### 1.1 设计理念

采用 **"活力极简"** 风格，核心原则：
- 信息层级清晰，一眼看到核心数据
- 动作引导明确，减少用户思考成本
- 视觉有活力感，符合健身场景调性

## 四、各页面设计要点

### 4.1 首页

**核心元素**：
- 连续打卡激励卡（渐变背景）
- 今日计划卡片（带进度）
- 快捷入口网格（3列）
- 最近训练列表

**交互逻辑**：
- 点击"开始训练"进入训练打卡页
- 快捷入口跳转对应功能页
- 训练记录项点击查看详情

### 4.2 训练计划列表页

**核心元素**：
- 计划卡片（图标+名称+进度）
- 新建计划按钮
- 空状态引导

**交互逻辑**：
- 卡片点击进入详情
- 滑动卡片显示编辑/删除
- 新建按钮进入编辑页

### 4.3 训练打卡页

**核心元素**：
- 计时器（实时显示）
- 当前动作卡片
- 重量/次数步进器
- 组数进度指示

**交互逻辑**：
- +/- 按钮调整数值
- "完成本组"记录数据并跳转下一组
- 退出训练弹窗确认

### 4.4 训练记录页

**核心元素**：
- 日历视图/列表视图切换
- 日历组件（训练日标记）
- 训练摘要卡片

**交互逻辑**：
- 点击日期显示当日记录
- 左右滑动切换月份
- 列表滚动加载更多

### 4.5 身体数据页

**核心元素**：
- 数据类型切换标签
- 当前数值概览
- 趋势折线图
- 历史数据列表

**交互逻辑**：
- 切换体重/体脂/BMI
- 图表触摸显示详情
- 时间范围切换

### 4.6 个人中心页

**核心元素**：
- 用户信息卡片
- 功能菜单列表
- 退出登录按钮

**交互逻辑**：
- 点击头像编辑信息
- 菜单项跳转对应页面
- 退出登录二次确认

## 五、页面跳转流程

```
登录页 -> 首页
  ├── 快捷入口 -> 训练计划编辑页 / 身体数据页
  ├── 今日计划 -> 训练打卡页 -> 训练完成弹窗
  └── Tab切换 -> 计划列表 / 训练记录 / 个人中心
      ├── 计划列表 -> 计划详情 -> 训练打卡页
      ├── 训练记录 -> 训练详情
      └── 个人中心 -> 身体数据页 / 设置页
```

## 六、组件复用规范

### 基础组件

| 组件 | 用途 |

| loading | 加载状态 |

| 组件 | 用途 |
|------|------|
| plan-card | 训练计划卡片 |
| action-item | 训练动作项 |
| record-item | 训练记录项 |
| calendar | 日历组件 |
| chart | 图表组件 |

## 七、动画规范

| 场景 | 动画 | 时长 |
|------|------|------|
| 页面进入 | 从右滑入 | 300ms |
| 弹窗出现 | 底部滑入 | 250ms |
| 打卡成功 | 缩放弹跳 | 500ms |
| 按钮点击 | 缩放0.95 | 100ms |

## 八、反馈机制

| 操作 | 反馈 |
|------|------|
| 打卡成功 | 振动 + "打卡成功" toast |
| 数据保存 | "保存成功" toast |
| 删除操作 | 二次确认弹窗 |
| 网络错误 | "网络异常，请重试" toast |
