## 一、功能模块总览

| 优先级 | 模块 | 说明 | 依赖 |
|--------|------|------|------|
| **P0** | 用户认证 | 微信登录，系统基础能力 | 无 |
| **P0** | 首页 | 今日概览、快捷入口 | 用户认证 |
| **P0** | 训练计划 | 创建/管理训练计划 | 用户认证 |
| **P0** | 训练记录 | 打卡与历史查询 | 训练计划 |
| **P1** | 身体数据 | 数据录入与趋势展示 | 用户认证 |

---

## 二、用户认证模块

### 用户故事
> 作为一个新用户，我希望通过微信一键登录，这样我不需要记住额外的账号密码。

### 功能描述
- 微信授权登录
- 用户信息获取（昵称、头像）
- 登录态维护（Token）

### 验收标准
- [ ] 点击登录按钮调起微信授权
- [ ] 授权成功后自动创建用户记录
- [ ] Token有效期7天，过期自动续期

---

## 三、训练计划模块

### 用户故事
> 作为一个健身用户，我希望创建自己的训练计划，按天安排不同的训练内容。

### 功能描述
- 创建训练计划（名称、描述、训练天数）
- 添加训练动作（名称、组数、次数、休息时间）
- 编辑/删除计划
- 查看计划详情

### 验收标准
- [ ] 支持创建多个训练计划
- [ ] 每个计划可包含多个训练日
- [ ] 每个训练日可包含多个动作
- [ ] 支持编辑和删除操作

---

## 四、训练记录模块

### 用户故事
> 作为一个健身用户，我希望记录每次训练的完成情况，包括重量、次数等数据。

### 功能描述
- 训练打卡（记录每组完成情况）
- 训练历史查询（按日期筛选）
- 今日训练概览

### 验收标准
- [ ] 支持记录每组的重量和次数
- [ ] 自动计算训练时长
- [ ] 按日期展示训练记录

---

## 五、身体数据模块

### 用户故事
> 作为一个健身用户，我希望记录体重、体脂等身体数据，观察变化趋势。

### 功能描述
- 录入身体数据（体重、体脂率、BMI等）
- 数据趋势图表展示
- 历史数据查询

### 验收标准
- [ ] 支持录入多种身体指标
- [ ] 折线图展示数据趋势
- [ ] 支持按时间范围筛选
