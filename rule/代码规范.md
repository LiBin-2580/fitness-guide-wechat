# 代码规范

> 适用于前端（微信小程序）和后端（Express）的所有代码

---

## 一、命名规范

### 1.1 文件命名

| 类型 | 规范 | 示例 |
|------|------|------|
| JavaScript文件 | kebab-case | `user-controller.js`、`auth-middleware.js` |
| WXML文件 | kebab-case | `plan-card.wxml` |
| WXSS文件 | kebab-case | `plan-card.wxss` |
| SQL文件 | kebab-case | `create-users-table.sql` |
| 配置文件 | kebab-case | `db-config.js` |

### 1.2 变量命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 普通变量 | camelCase | `userName`、`planList` |
| 常量 | UPPER_SNAKE_CASE | `MAX_RETRY_COUNT`、`API_BASE_URL` |
| 布尔变量 | is/has/can前缀 | `isLoading`、`hasPermission` |
| 函数 | camelCase，动词开头 | `getUserInfo()`、`handleSubmit()` |
| 类/组件 | PascalCase | `PlanCard`、`UserService` |
| 私有变量 | _前缀 | `_cacheData`（尽量避免使用） |

### 1.3 CSS命名

采用 **BEM** 命名规范：

```css
/* Block */
.plan-card {}

/* Element */
.plan-card__title {}
.plan-card__content {}

/* Modifier */
.plan-card--active {}
.plan-card--disabled {}
```

### 1.4 数据库命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 表名 | snake_case，复数 | `users`、`training_records` |
| 字段名 | snake_case | `user_id`、`created_at` |
| 主键 | id | `id` |
| 外键 | 表名单数_id | `user_id`、`plan_id` |
| 时间字段 | xxx_at | `created_at`、`updated_at` |
| 布尔字段 | is_xxx | `is_active`、`is_deleted` |

### 1.5 API命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 路径 | kebab-case，复数 | `/api/v1/training-plans` |
| 查询参数 | camelCase | `pageNum`、`pageSize` |
| 请求体字段 | camelCase | `userId`、`planName` |
| 响应体字段 | camelCase | `createdAt`、`exerciseCount` |

## 二、代码风格

### 2.1 JavaScript通用规范

```javascript
// ✅ 使用 const/let，不用 var
const MAX_COUNT = 10;
let currentCount = 0;

// ✅ 使用模板字符串
const message = `用户 ${userName} 创建成功`;

// ✅ 使用解构赋值
const { name, age } = userInfo;

// ✅ 使用箭头函数（回调场景）
const newList = list.map(item => item.id);

// ✅ 使用 async/await 处理异步
async function fetchData() {
  try {
    const result = await api.getData();
    return result;
  } catch (error) {
    console.error('获取数据失败:', error);
    throw error;
  }
}

// ❌ 避免使用 ==，使用 ===
if (value === null) { }

// ❌ 避免魔法数字
if (status === 1) { }  // ❌
if (status === STATUS_ACTIVE) { }  // ✅
```

```javascript

  // ...

try {
  await saveData(data);

  // ❌ 不要吞掉错误
  // ✅ 要么抛出，要么处理
```

## 三、代码质量

### 3.1 减少嵌套

```javascript
// ❌ 嵌套过深
function processUser(user) {
  if (user) {
    if (user.age >= 18) {
      if (user.isActive) {
        // do something
      }
    }
  }
}

// ✅ 提前返回
function processUser(user) {
  if (!user) return;
  if (user.age < 18) return;
  if (!user.isActive) return;
  
  // do something
}
```

### 3.2 避免重复代码

```javascript
// ❌ 重复逻辑
function getActivePlans(userId) {
  return db.query('SELECT * FROM plans WHERE user_id = ? AND status = 1', [userId]);
}
function getInactivePlans(userId) {
  return db.query('SELECT * FROM plans WHERE user_id = ? AND status = 0', [userId]);
}

// ✅ 合并为通用函数
function getPlansByStatus(userId, status) {
  return db.query('SELECT * FROM plans WHERE user_id = ? AND status = ?', [userId, status]);
}
```

### 3.3 常量管理

```javascript
// ✅ 集中定义常量
// constants/status.js
module.exports = {
```

## 四、错误处理

### 4.1 统一错误格式

```javascript
// 自定义错误类
class AppError extends Error {
  constructor(message, code = 500) {
    super(message);
    this.code = code;
    this.isOperational = true;
  }
}

// 使用
throw new AppError('用户不存在', 404);
throw new AppError('参数错误: name不能为空', 400);
```

### 4.2 错误日志

```javascript
// ✅ 记录足够的上下文信息
logger.error('创建计划失败', {
  userId: req.user.id,
  planData: req.body,
  error: error.message,
  stack: error.stack,
});

// ❌ 不要只打印错误对象
console.log(error);  // ❌
```

## 五、安全规范

### 5.1 输入验证

```javascript
// ✅ 所有用户输入都要验证
const { name, difficulty } = req.body;

if (!name || typeof name !== 'string' || name.length > 100) {
  throw new AppError('计划名称无效', 400);
}

if (!['easy', 'medium', 'hard'].includes(difficulty)) {
  throw new AppError('难度等级无效', 400);
}
```

### 5.2 SQL防护

```javascript
// ❌ 拼接SQL（有注入风险）
const sql = `SELECT * FROM users WHERE name = '${name}'`;

// ✅ 使用参数化查询
const sql = 'SELECT * FROM users WHERE name = ?';
db.query(sql, [name]);
```

### 5.3 敏感信息

```javascript
// ❌ 不要在代码中硬编码密钥
const SECRET = 'my-secret-key';

// ✅ 使用环境变量
const SECRET = process.env.JWT_SECRET;

// ❌ 不要在响应中返回敏感字段
res.json({ user: { id: 1, password: 'xxx' } });

// ✅ 过滤敏感字段
const { password, ...safeUser } = user;
res.json({ user: safeUser });
```
