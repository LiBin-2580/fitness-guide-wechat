# 后端开发规范

> Express后端开发规范

---

## 一、目录结构

```
server/
├── config/         # 配置文件
├── middleware/      # 中间件（auth、validator、errorHandler）
├── routes/         # 路由定义
├── controllers/    # 控制器（处理请求）
├── services/       # 业务逻辑层
├── models/         # 数据模型（数据库操作）
├── utils/          # 工具函数
├── app.js          # 应用入口
├── package.json    # 依赖配置
└── .env            # 环境变量（不提交Git）
```

---

## 二、分层职责

```
请求 → Route → Controller → Service → Model → 数据库
```

| 层级 | 职责 | 注意事项 |
|------|------|----------|
| Route | 定义URL和HTTP方法 | 只做路由绑定，不含逻辑 |
| Controller | 接收参数，调用Service，返回响应 | 处理req/res，不含业务逻辑 |
| Service | 业务逻辑处理 | 不直接操作数据库，组合Model |
| Model | 数据库操作 | 只做CRUD，不含业务判断 |

## 三、路由规范

```javascript
// RESTful风格
GET    /api/v1/plans          // 获取列表
GET    /api/v1/plans/:id      // 获取详情
POST   /api/v1/plans          // 创建
PUT    /api/v1/plans/:id      // 更新
DELETE /api/v1/plans/:id      // 删除

// 特殊操作
GET    /api/v1/records/today  // 获取今日记录
POST   /api/v1/auth/login     // 登录
```

## 四、统一响应格式

```javascript
// utils/response.js
module.exports = {
  success(res, data = null, statusCode = 200) {
    res.status(statusCode).json({ code: 0, message: 'success', data });
  },
  error(res, message = '服务器错误', statusCode = 500) {
    res.status(statusCode).json({ code: statusCode, message, data: null });
  },
};
```

## 五、自定义错误类

```javascript
class AppError extends Error {
  constructor(message, statusCode = 500) {
    super(message);
    this.statusCode = statusCode;
    this.isOperational = true;
  }
}
module.exports = AppError;

// 使用
throw new AppError('计划不存在', 404);
```

## 六、中间件

### 认证中间件

```javascript
// middleware/auth.js
module.exports = function(req, res, next) {
  const token = req.headers.authorization?.replace('Bearer ', '');
  if (!token) return error(res, '未提供token', 401);
  
  try {
    req.user = jwt.verify(token, process.env.JWT_SECRET);
    next();
  } catch (err) {
    error(res, 'token无效', 401);
  }
};
```

### 错误处理中间件

```javascript
// middleware/errorHandler.js
module.exports = function(err, req, res, next) {
  logger.error('服务器错误', { url: req.url, error: err.message });
  const statusCode = err.statusCode || 500;
  const message = err.isOperational ? err.message : '服务器内部错误';
  error(res, message, statusCode);
};
```
